# 缺陷排查修复

## 何时使用

适用于：

- API 返回错误数据或状态码异常
- 登录、鉴权、会话相关故障
- mapper、service、controller 行为回归
- 环境或配置导致的行为不一致

## 排查流程

复制清单：

```text
缺陷处理进度
- [ ] 明确预期行为与实际行为
- [ ] 定位故障层（controller/service/mapper/config）
- [ ] 实施最小且安全的修复
- [ ] 验证改动文件（lint/可用测试）
- [ ] 行为变化时更新文档风险与备注
```

## 排查规则

- 从问题接口或模块入手，避免全局泛扫。
- 结论必须基于当前代码和配置事实。
- 除非修复必须，否则不改变 API 契约。

## 修复规则

- 最小化影响面，避免无关重构。
- 临时行为调整优先配置开关，不做硬编码。
- 异常处理要清晰，面向用户提示要明确。

## 输出内容

返回：

1. 根因
2. 修复摘要（含文件路径）
3. 验证状态
4. 剩余风险与后续项
